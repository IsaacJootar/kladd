# KLADD — Project Overview

## Core Principle
Verify once. Prove everywhere.

## What Kladd Is
Kladd is a reusable proof, verification, and claim exchange platform built around the idea that users should not repeatedly upload raw documents to organizations.

Instead:
- users store and verify evidence once,
- organizations request operational truths,
- users approve requests using a Security PIN,
- Kladd issues time-bound claims.

## What Kladd Is NOT
Kladd is NOT:
- a raw file sharing app
- a Google Drive clone
- a document forwarding system
- unrestricted identity exchange

Kladd is:
- a reusable verification network
- a claim exchange system
- a truth delivery platform

## Core Concepts

### Evidence
Protected records uploaded or linked by users.

### Truth
Operational verification outputs such as identity_verified, degree_verified, address_verified.

### Claim
A time-bound package containing approved truths.

### Claim Session
A complete verification interaction: request → approval → claim issuance → retrieval → expiry.

### Consent Approval
A security event where the user reviews a claim request and confirms release using their Security PIN.

## Main Product Goals
- eliminate repeated uploads
- minimize document exposure
- create reusable verification
- enable controlled trust exchange
- provide auditability and visibility
