# KLADD — Design & UI Specification

## Core UI Philosophy
Kladd must feel simple, calm, smooth, modern, trustworthy, and lightweight.

Users should feel:
"I am approving proofs and organizing my records."

Users should not feel:
"I am managing identity infrastructure."

## Design Principles
- minimal cognitive load
- human language only
- large clean cards
- soft spacing
- clear typography
- mobile-first approval flows

## User Navigation
- Home
- My Records
- Requests
- Proofs
- Security

## Dashboard Cards
- Identity Status
- Pending Requests
- Active Proofs
- Recent Activity

## My Records Page
Use record cards, not a file manager.

Example:
Passport
Verified ✓
Expires: Jan 2029

## Requests Page
Request cards must clearly show:
- requester
- requested proofs
- purpose
- duration
- Approve/Deny buttons

## Approval Experience
Approval must feel secure and focused:
- centered modal
- dark overlay
- Security PIN input
- short explanation
- no clutter

## Proofs Page
Shows active, expired, and revoked proofs.
Expired/revoked proofs must show metadata only.

## QR/PIN UX
Maximum 2 taps:
Generate Verification → show large QR or exchange PIN.

## Organization UI
Professional, simple, operational:
- Dashboard
- Requests
- Active Proofs
- Expired Proofs
- Verification Logs
- Developers
- Settings

## Visual Style
- modern
- minimal
- soft enterprise
- trustworthy

Recommended colors:
- white/light gray base
- deep blue or indigo primary
- soft green success
- amber warning
- muted red danger
